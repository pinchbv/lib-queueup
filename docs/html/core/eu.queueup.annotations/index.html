AuthAware

@Target(allowedTargets = [AnnotationTarget.FUNCTION])
annotation class AuthAware

Marks an endpoint that does not require authentication but adapts its behavior based on whether a session is present. Unauthenticated callers are treated as guests; authenticated callers see responses scoped to their account.

NoAuthentication
Link copied to clipboard
@Target(allowedTargets = [AnnotationTarget.FUNCTION])
annotation class NoAuthentication

Marks an endpoint that is fully public. The backend does not associate the call with any user — authenticated or otherwise — and the response is identical regardless of session state.

NotAnEndpoint
Link copied to clipboard
@Target(allowedTargets = [AnnotationTarget.FUNCTION])
annotation class NotAnEndpoint

Marks a member that is not an HTTP endpoint — typically a local store check or a reactive state observer. Such members are exempt from the auth-posture annotation rule because the "requires / does not require / auth-aware" distinction only meaningfully describes network calls.

RequiresAuthentication
Link copied to clipboard
@Target(allowedTargets = [AnnotationTarget.FUNCTION])
annotation class RequiresAuthentication

Marks an endpoint that requires the user to be authenticated. Calls made without a valid session fail with eu.queueup.errors.Error.Unauthorized and the SDK signals the host via eu.queueup.session.auth.Auth.authenticationRequired.
